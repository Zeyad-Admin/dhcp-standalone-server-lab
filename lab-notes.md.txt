## üèóÔ∏è Implementation Steps

1. Created virtual machines for DC-Z, DHCP-Z, and CL-Z1  
2. Configured static IP addresses for infrastructure servers  
3. Joined DHCP-Z to the corp.mog domain  
4. Installed the DHCP Server role on DHCP-Z  
5. Exported DHCP configuration from DC-Z  
6. Imported DHCP configuration to DHCP-Z  
7. Disabled DHCP service on DC-Z  
8. Authorized DHCP-Z in Active Directory  
9. Renewed client IP configuration to obtain address from DHCP-Z  
10. Verified connectivity and DNS resolution  

### DHCP Configuration Migration

The existing DHCP configuration was exported from the domain controller (DC-Z) using PowerShell and imported into the dedicated DHCP server (DHCP-Z).

This approach mirrors real-world infrastructure practices where DHCP services are migrated to standalone servers for scalability, security, and fault isolation.
